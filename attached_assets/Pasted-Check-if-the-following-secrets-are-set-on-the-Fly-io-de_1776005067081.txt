Check if the following secrets are set on the Fly.io deployment. If any are missing, set them automatically:

1. SESSION_SECRET — generate with: openssl rand -hex 32
2. ANTHROPIC_API_KEY — required for the AI/CBA Assistant tab to function

Run this check and fix script:

# Check and set SESSION_SECRET
SECRETS=$(flyctl secrets list 2>/dev/null || echo "")

if echo "$SECRETS" | grep -q "SESSION_SECRET"; then
  echo "✅ SESSION_SECRET is set"
else
  echo "⚠️ SESSION_SECRET missing — setting now..."
  fly secrets set SESSION_SECRET="$(openssl rand -hex 32)"
fi

# Check ANTHROPIC_API_KEY
if echo "$SECRETS" | grep -q "ANTHROPIC_API_KEY"; then
  echo "✅ ANTHROPIC_API_KEY is set"
else
  echo "⚠️ ANTHROPIC_API_KEY missing — add it with:"
  echo 'fly secrets set ANTHROPIC_API_KEY="your-key-here"'
fi

# Check SESSION_SECRET in app environment
if [ -z "$SESSION_SECRET" ]; then
  echo "⚠️ SESSION_SECRET not found in environment"
  export SESSION_SECRET=$(openssl rand -hex 32)
  echo "✅ SESSION_SECRET generated for local use"
fi

REQUIRED ENVIRONMENT VARIABLES — ensure these exist in both .env and Fly.io secrets:

SESSION_SECRET=<32-byte hex string generated by openssl rand -hex 32>
ANTHROPIC_API_KEY=<your Anthropic API key from console.anthropic.com>
DATABASE_URL=<your database connection string if applicable>
NODE_ENV=production

IN YOUR APP CODE — ensure session middleware uses SESSION_SECRET:

// Express session setup
app.use(session({
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  cookie: {
    secure: process.env.NODE_ENV === 'production',
    httpOnly: true,
    maxAge: 24 * 60 * 60 * 1000 // 24 hours
  }
}))

// Anthropic AI client setup
import Anthropic from '@anthropic-ai/sdk'
const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY
})

IN fly.toml — ensure this is present:

[env]
  NODE_ENV = "production"

[deploy]
  release_command = "npm run migrate"

NEVER hardcode secrets in source code. Always use process.env variables. Verify secrets are loaded before starting the server — if SESSION_SECRET or ANTHROPIC_API_KEY is undefined at startup, throw an error and exit.

Add this startup check to your server entry file (index.js or server.js):

const required = ['SESSION_SECRET', 'ANTHROPIC_API_KEY']
for (const key of required) {
  if (!process.env[key]) {
    console.error(`❌ Missing required environment variable: ${key}`)
    process.exit(1)
  }
}
console.log('✅ All required environment variables are set')
