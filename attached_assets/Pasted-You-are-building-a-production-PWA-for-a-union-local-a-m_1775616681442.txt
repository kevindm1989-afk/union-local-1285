You are building a production PWA for a union local, a mobile-first steward management tool deployed on Fly.io with a Neon PostgreSQL database.

## Stack (do not change)
- Monorepo: pnpm workspaces
- Frontend: React + Vite + shadcn/ui + TanStack Query + Wouter
- Backend: Express 5 + TypeScript
- Database: Neon PostgreSQL (serverless) + Drizzle ORM
- Validation: Zod (v4) + drizzle-zod
- API codegen: Orval (OpenAPI spec)
- Build: esbuild (CJS bundle)
- Node: 24, TypeScript: 5.9
- Deploy: Fly.io (Dockerfile already configured)
- AI: Anthropic Claude API (CBA Q&A assistant)

## Database (Neon)
Use `DATABASE_URL` environment variable (already set). Neon is serverless — always use connection pooling via `@neondatabase/serverless` driver with `{ ssl: true }`. Do not use `pg` directly.

## What already exists
- Member CRUD (name, employee_id, department, classification, phone, email, join_date, is_active, notes)
- Grievance tracking (Steps 1–4, statuses: open | pending_response | pending_hearing | resolved | withdrawn)
- Announcements/bulletins (categories: general | urgent | contract | meeting | action)
- Dashboard summary stats
- Basic RBAC (role-configurable)
- Claude AI CBA assistant
- Bottom tab nav: Dashboard, Members, Grievances, Bulletins

## Coding Rules
- Always use Drizzle ORM — never raw SQL
- All new tables need a migration file under db/migrations/
- All new API routes need corresponding OpenAPI spec entries and Orval codegen run
- All sensitive member fields (phone, email, organizing_notes) must never appear in grievance API responses
- Role checks on every protected route: throw 403 if role insufficient
- Never expose audit_log or engagement data to "member" role
- All dates stored as UTC — display in the local timezone configured by the deployment
- TypeScript strict mode — no `any` types
- Error responses: { error: string, code: string } format consistently

## Labour Law Context
- Grievance deadlines are governed by the collective agreement — always surface due_date prominently
- Duty to accommodate — add accommodation_request boolean field on grievances
- Privacy compliance: member personal data access must be logged (covered by audit_log above)
- Grievance step deadlines are configurable — do not hardcode; read from a local_settings table
- All legal references in the UI should use neutral language (e.g. "collective agreement" not jurisdiction-specific statute names)
