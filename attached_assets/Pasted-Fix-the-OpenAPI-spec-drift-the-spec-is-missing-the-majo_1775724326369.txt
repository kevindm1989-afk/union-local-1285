Fix the OpenAPI spec drift — the spec is missing the majority of API routes 
causing Orval to generate zero typed hooks for them. All frontend calls to 
these endpoints are currently untyped raw fetch calls.

File: lib/api-spec/openapi.yaml

## Instructions
Do not change any existing spec entries. Only ADD the missing route definitions 
below. After adding all routes run:
pnpm --filter @workspace/api-spec run codegen
to regenerate all Orval hooks and Zod schemas.

## Standard response schemas to apply to all routes

### Success responses
200/201:
{
  data: [route-specific object or array]
  meta: (for list endpoints only)
    {
      total: integer
      limit: integer
      offset: integer
    }
}

### Error responses (apply to all routes)
400: { error: string, code: 'VALIDATION_ERROR' }
401: { error: string, code: 'UNAUTHORIZED' }
403: { error: string, code: 'FORBIDDEN' }
404: { error: string, code: 'NOT_FOUND' }
409: { error: string, code: 'CONFLICT' }
422: { error: string, code: 'VALIDATION_ERROR' }
429: { error: string, code: 'RATE_LIMITED' }
500: { error: string, code: 'INTERNAL_ERROR' }

## After adding all routes
